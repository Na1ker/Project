# TradeTracker v1.4 — выразительное появление дашборда

## Цель (Objective)

Дашборд открывается функционально, но невыразительно: элементы просто оказываются на месте (лёгкий rise-in есть, но он почти незаметен). Первое, что видит трейдер каждый раз, заходя в трекер, — главный экран, и он заслуживает момента «вау». Задача — сделать появление дашборда при загрузке страницы выразительным и приятным, оставаясь в рамках вкуса (быстро, по делу, без раздражения при повторных заходах).

Это чисто визуальное улучшение поверх существующего дашборда: та же вёрстка и данные, но элементы эффектно проявляются оркестрованным каскадом при полной загрузке главной страницы. Переходы по навигации (Сделки/Анализ/Настройки) не затрагиваются — там скорость важнее эффекта.

## Требования (Requirements)

### Обязательные (Must-have)

1. При **полной загрузке дашборда** (`/`) элементы появляются выразительным оркестрованным каскадом, а не все разом. Порядок появления сверху вниз по смыслу: сначала блок капитала и переключатель периода, затем карточка кривой капитала, затем панель метрик, затем блок открытых позиций.
2. Каждый элемент входит комбинацией **прозрачность 0→1 + сдвиг вверх** (с ~16–20px) и лёгкого масштаба (с ~0.98, никогда не с 0 — ничто не появляется из ниоткуда). Движение только на `transform`/`opacity` (GPU).
3. Кривая капитала (график) при первом появлении **прорисовывается** приятно: либо клип-ридил слева направо, либо плавное проявление серии — не резкий показ готового графика. Свечных/тяжёлых перерисовок при этом не делать (лёгкий эффект поверх готовой отрисовки).
4. Тайминги в рамках вкуса: каждый элемент ≤ ~450мс, шаг каскада между блоками 60–90мс, весь оркестр завершается **менее чем за ~900мс** — дашборд не заставляет ждать. Сильные ease-out кривые (существующие `--ease-out-strong`), никакого ease-in.
5. Анимация играет **при полной загрузке страницы** (первый заход, F5, переоткрытие вкладки), но **не** при клиентских переходах на дашборд из меню (Сделки → Дашборд не должно каждый раз проигрывать оркестр). Реализация должна отличать полную загрузку от client-side навигации.
6. Анимация **не блокирует взаимодействие**: переключатель периода, ссылки и позиции кликабельны с первого момента; каскад — чисто декоративный слой.
7. **Reduced-motion**: при `prefers-reduced-motion: reduce` каскад заменяется мгновенным/мягким проявлением без движения и без прорисовки графика (появление за один короткий fade).
8. Баннер ошибки синхронизации (когда есть) появляется сразу, вне каскада — важная информация не ждёт анимацию.

### Желательные (Nice-to-have, НЕ входит в v1.4)

- Заставка-сплэш с логотипом на весь экран.
- Анимации появления на других страницах (Сделки/Анализ/Настройки).
- Анимация при client-side переходах между страницами.
- Разное поведение для «самого первого визита» против обычных заходов.
- Микро-интеракции наведения на карточки метрик.

## Ограничения (Constraints)

- Стек прежний (Next.js, React, Tailwind, CSS). Без новых анимационных библиотек — только CSS-анимации/transitions и, при необходимости, минимальный JS для оркестрации задержек. Существующий motion-фундамент (`--ease-out-strong`, классы `rise-in`/`fade-in` в globals.css) переиспользуется и при необходимости расширяется.
- Только `transform`/`opacity` в анимациях (производительность; см. motion-принципы проекта).
- Прорисовка графика — лёгкий CSS-эффект поверх готового canvas lightweight-charts (например clip-path/opacity на контейнере), без вмешательства в саму библиотеку и без повторных setData.
- Не трогать логику данных, синхронизацию, другие страницы.

## Крайние случаи (Edge cases)

- **Онбординг (нет ключей)** → дашборд редиректит на `/settings`; интро-анимация не должна мелькать до редиректа (либо не запускаться, либо редирект происходит раньше её видимой фазы).
- **Данные ещё грузятся** (fetch метрик/позиций не завершён) → каркас блоков появляется каскадом, содержимое подставляется по готовности без повторного проигрывания каскада.
- **Нет снапшотов капитала / нет сделок за период** → пустые состояния появляются в том же каскаде, эффект прорисовки графика на пустом графике не выглядит сломанным (просто fade).
- **Client-side переход на дашборд** (из меню) → каскад НЕ играет; дашборд показывается сразу.
- **Повторный ре-рендер дашборда** (смена периода, обновление данных каждые 30с) → каскад НЕ переигрывается; играет строго один раз за загрузку страницы.
- **reduced-motion** → без движения и прорисовки, только короткий fade; интерактив доступен сразу.
- **Баннер ошибки BingX** → появляется немедленно, не участвует в декоративной задержке.

## Definition of Done

- [ ] Обновляю страницу на `/` → блоки дашборда появляются красивым каскадом сверху вниз (капитал → график → метрики → позиции), весь оркестр < ~900мс.
- [ ] Кривая капитала при загрузке приятно прорисовывается, а не показывается рывком.
- [ ] Кликаю переключатель периода или ссылку во время анимации — реагирует сразу, ничего не заблокировано.
- [ ] Перехожу Сделки → Дашборд через меню — каскад НЕ играет, дашборд сразу на месте.
- [ ] Меняю период / жду автообновления — каскад НЕ переигрывается.
- [ ] Включаю «Уменьшить движение» в системе → появление без движения и прорисовки, только мягкий fade, всё сразу кликабельно.
- [ ] Баннер ошибки синхронизации (если есть) виден сразу, не ждёт анимацию.
- [ ] Анимации используют только transform/opacity, сильные ease-out; на глаз плавно, без рывков.
