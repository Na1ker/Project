# TradeTracker v1.3 — журнал заметок и разметка графика (датасет для LLM)

## Цель (Objective)

Заметка к сделке сейчас — одно текстовое поле, а мысли трейдера на график никак не попадают. v1.3 делает два шага к «дневнику трейдера, который читает машина»:

1. Заметки становятся **журналом**: каждая сохранённая запись добавляется в список с временем, поле очищается — к сделке копится история мыслей.
2. На графике сделки появляется **инструмент рисования** (v1 — трендовая линия): свои уровни и линии сохраняются вместе со сделкой, привязаны к цене и времени.

Всё сохраняется как структурированный датасет (записи журнала + фигуры разметки + параметры и результат сделки) — база, на которой следующая версия сможет запускать LLM-анализ сетапов («какие мои идеи зарабатывают, какие сливают»). В v1.3 датасет только копится, никакого анализа и внешних API.

## Требования (Requirements)

### Обязательные (Must-have)

**Журнал заметок**
1. Заметки к сделке — список записей: каждая с текстом и временем создания, новые сверху. Таблица `notes` заменяется/расширяется до множественных записей на `trade_key` (существующая заметка мигрирует как первая запись журнала — не теряется).
2. После нажатия «Сохранить заметку» запись появляется в списке, **поле очищается**. Пустой текст не сохраняется.
3. Запись можно удалить (маленькая кнопка на записи, без подтверждения — записи короткие).
4. В списке сделок пометка «есть заметка» остаётся (есть хотя бы одна запись журнала).

**Разметка графика**
5. На странице сделки под графиком (в том же блоке-карточке) — маленькая кнопка-иконка инструментов. Клик открывает меню выбора инструмента; в v1 один пункт — **«Трендовая линия»**.
6. Рисование: выбрал инструмент → первый клик по графику ставит первую точку, второй — вторую, линия готова. Линия привязана к **цене и времени** (не к пикселям): при зуме/панораме графика едет вместе со свечами.
7. Нарисованные линии сохраняются в базе с привязкой к сделке (`trade_key`) сразу после постановки второй точки; при повторном открытии страницы — восстанавливаются.
8. Линию можно удалить: клик по линии выделяет её, появляется кнопка удаления (или клавиша Delete/Backspace).
9. Лимитов на число линий нет (разумно до десятков); линии рисуются только когда график доступен.
10. Пока инструмент активен — курсор-перекрестие и подсказка («кликни первую точку…»); Escape отменяет рисование; выход из режима после постановки линии.

**Датасет**
11. Структура в базе позволяет собрать по сделке единый JSON: параметры сделки (инструмент, направление, плечо, цены, PnL), записи журнала с временем, фигуры разметки с координатами (тип, точки цена/время, цвет). Формат зафиксирован в коде (типы TypeScript) — это контракт будущего LLM-анализа.

**Общее**
12. Дизайн в существующей системе (тёмная тема, палитра, pressable, ≤300мс); иконка инструментов ненавязчивая, в стиле остальных контролов.
13. Никаких внешних запросов: всё локально.

### Желательные (Nice-to-have, НЕ входит в v1.3)

- Другие инструменты: горизонтальный уровень, прямоугольник, текст на графике, фибо, стрелки.
- Выбор цвета/толщины линии; редактирование концов существующей линии перетаскиванием.
- LLM-анализ датасета (следующая версия).
- Разметка на кривой капитала дашборда.

## Ограничения (Constraints)

- Стек прежний. Рисование — слой поверх lightweight-charts (SVG/canvas overlay) с конвертацией координат через API графика (`timeScale().coordinateToTime/timeToCoordinate`, `priceToCoordinate/coordinateToPrice`).
- Миграция заметок обязана быть безопасной: существующая заметка пользователя (реальная, про HYPE) не теряется и видна как первая запись журнала.
- Datasets: без внешних сервисов, всё в SQLite.

## Крайние случаи (Edge cases)

- **Существующая заметка** → после миграции видна как запись журнала со своим временем сохранения.
- **График недоступен** (нет свечей) → кнопка инструментов скрыта/неактивна, сохранённые линии не рисуются, но и не теряются.
- **Точка линии за пределами видимого диапазона** после зума → линия корректно продолжается за край или частично видима; ничего не падает.
- **Клик по графику без активного инструмента** → ничего не происходит (обычное поведение графика не ломается: зум/панорама работают как раньше).
- **Первая точка поставлена, пользователь ушёл со страницы или нажал Escape** → незавершённая линия не сохраняется.
- **Удаление записи журнала/линии** → исчезает сразу, после перезагрузки не возвращается.
- **Смена таймфрейма графика** (другая длительность сделки при повторном открытии) → линии привязаны к абсолютному времени, отображаются корректно на любом таймфрейме.
- **Открытая сделка** → журнал и разметка работают так же; после закрытия сделки заметки/линии переезжают вместе с миграцией trade_key (существующий механизм переноса заметок расширяется на журнал и линии).

## Definition of Done

- [ ] Пишу запись в поле → «Сохранить» → запись появляется в списке с временем, поле пустое; пишу вторую — обе в списке, новые сверху.
- [ ] Моя старая заметка про HYPE видна как первая запись журнала после обновления.
- [ ] После перезапуска сервера обе записи на месте; удалённая запись не возвращается.
- [ ] На странице сделки под графиком есть кнопка-иконка; клик → меню с «Трендовая линия»; двумя кликами рисую линию на графике.
- [ ] Линия привязана к цене/времени: зум и панорама двигают её вместе со свечами.
- [ ] Перезагружаю страницу — линия на месте; выделяю и удаляю — исчезает навсегда.
- [ ] Пометка заметки в списке сделок работает при наличии хотя бы одной записи.
- [ ] Клики по графику без инструмента ведут себя как раньше (зум/панорама не сломаны).
- [ ] В коде зафиксирован тип «датасет сделки» (JSON: сделка + журнал + фигуры), и его можно получить по API для будущего LLM-анализа.
