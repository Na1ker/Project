# TradeTracker v1.2 — Монте-Карло и статистический анализ сделок

## Цель (Objective)

Трекер уже собирает полную историю сделок с BingX (89+ закрытых позиций), но пока только показывает её. Следующий шаг — превратить историю в понимание: «если я продолжу торговать так же — что будет с депозитом?», «какая просадка для меня нормальна, а какая — сигнал, что что-то сломалось?», «какой у моей торговли реальный профиль?».

Добавляем страницу **«Анализ»**: Монте-Карло симуляция будущего депозита на основе реальных сделок пользователя, метрики риска (просадки, серии убытков) и статистический профиль торговли (распределение результатов, экспектанси, паттерны по времени). Всё считается локально по данным из собственной базы — никаких внешних сервисов.

## Требования (Requirements)

### Обязательные (Must-have)

**Страница и выборка**
1. Новая страница `/analysis` («Анализ») с пунктом в шапке между «Сделки» и «Настройки».
2. Выборка данных: закрытые сделки выбранного периода — переключатель **7 / 30 / 90 / 180 дней / Всё время** (по умолчанию «Всё время»). Скрытые инструменты исключаются. Результат сделки = чистый PnL (цифра биржи `net_profit`).
3. Порог достоверности: при **менее 20 сделках** в выборке страница показывает состояние «мало данных для анализа: N из 20 сделок» и не рисует симуляцию (частичные метрики профиля можно показать с пометкой «мало данных»).

**Монте-Карло симуляция**
4. Метод: bootstrap — случайная выборка с возвращением из чистых PnL реальных сделок выборки. Горизонт — **N будущих сделок** (по умолчанию 100, настраивается: 50/100/200/500). Число прогонов — **не менее 1000** (по умолчанию 1000, настраивается: 1000/5000/10000).
5. Стартовый капитал симуляции = текущий капитал аккаунта (последний снапшот equity).
6. Визуализация: **веер траекторий** депозита (не менее 50 случайных прогонов полупрозрачными линиями) + выделенные перцентили P5 / P50 (медиана) / P95 поверх. Цвета в палитре проекта (тёмно-красный/зелёный).
7. Карточки итогов симуляции: медианный итоговый капитал (и %), лучший сценарий (P95), худший (P5), вероятность закончить горизонт в минусе, вероятность просадки ≥25% и ≥50% в течение горизонта.
8. Симуляция пересчитывается по кнопке «Пересчитать» и при смене периода/параметров; расчёт с 10000 прогонов не блокирует интерфейс (веб-воркер или чанки) и занимает не более ~2 секунд на M-серии Mac.

**Риск-метрики (по историческим сделкам выборки)**
9. Максимальная историческая серия убыточных сделок подряд и максимальная серия прибыльных; ожидаемая максимальная серия убытков на горизонте N сделок (из симуляции).
10. Максимальная историческая просадка по кривой капитала выборки (в % и USDT) и её длительность.

**Профиль торговли**
11. Гистограмма распределения чистых PnL сделок (бины автоматически, убыточные — красные, прибыльные — зелёные).
12. Карточки: экспектанси (средний чистый PnL на сделку), стандартное отклонение результата, отношение среднего выигрыша к среднему проигрышу, доля прибыльных.
13. Разбивка по времени: винрейт и суммарный PnL по дням недели и по 4-часовым интервалам времени входа (локальная таймзона). Ячейки с < 3 сделками помечаются как малодостоверные (приглушённый цвет).

**Общее**
14. Все вычисления — на клиенте или в API-роуте из локальной SQLite; внешних запросов нет. Страница работает офлайн (данные уже в базе).
15. Дизайн — в существующей системе (тёмная тема, карточки, палитра, motion-фундамент: rise-in каскад, pressable кнопки, ≤300мс).

### Желательные (Nice-to-have, НЕ входит в v1.2)

- Симуляция с учётом размера позиции/риска на сделку (% депозита вместо абсолютных PnL).
- Фильтр анализа по инструменту или направлению (long/short).
- Анализ заметок «почему зашёл» и корреляция причин с результатом.
- Экспорт отчёта (PDF/картинка).
- Сравнение периодов («мой сентябрь против октября»).

## Ограничения (Constraints)

- Стек прежний: Next.js + TypeScript + node:sqlite, графики — lightweight-charts, где уместно, иначе лёгкий SVG/canvas без новых тяжёлых зависимостей (никаких chart.js/d3 целиком).
- Монте-Карло — честный bootstrap без подгонки распределений; фиксируем random seed опционально не требуется.
- Расчёты не блокируют UI (требование 8); при 10000 прогонов × 500 сделок допускается индикатор прогресса.
- Приватность: данные не покидают машину.

## Крайние случаи (Edge cases)

- **Меньше 20 сделок в выборке** → состояние «мало данных» (требование 3), никаких NaN и пустых графиков.
- **Все сделки прибыльные (или все убыточные)** → метрики считаются корректно: серия убытков = 0, отношение выигрыш/проигрыш — «∞»/«—» как в статистике дашборда; симуляция работает.
- **Один инструмент даёт 90% сделок** → анализ честно считает по всем; никакой нормализации молча не делается (упомянуть в подписи «на основе N сделок»).
- **Экстремальный выброс** (одна сделка −655 USDT из серии BTC) → гистограмма не схлопывается: крайние бины расширяются или помечаются, веер симуляции остаётся читаемым.
- **Стартовый капитал недоступен** (нет снапшотов equity) → симуляция стартует с суммы |PnL| выборки с пометкой, либо состояние «нет данных о капитале».
- **Смена периода во время расчёта** → предыдущий расчёт отменяется, стартует новый; результаты не смешиваются.
- **Скрытие инструмента в настройках** → при следующем открытии/пересчёте анализ его не учитывает.

## Definition of Done

- [ ] На странице «Анализ» с выборкой «Всё время» (89 сделок) вижу веер Монте-Карло с перцентилями P5/P50/P95 и карточки: медианный итог, лучший/худший сценарий, вероятность минуса, вероятности просадок ≥25%/≥50%.
- [ ] Смена горизонта (50→500) и числа прогонов (1000→10000) пересчитывает симуляцию; интерфейс не зависает.
- [ ] Медиана симуляции согласуется со здравым смыслом: примерно стартовый капитал + N × экспектанси (± случайность).
- [ ] Вижу свою реальную максимальную серию убытков, максимальную просадку, гистограмму PnL, экспектанси и разбивку по дням недели/времени суток.
- [ ] Переключение на «7 дн» с < 20 сделками показывает «мало данных», а не пустые графики.
- [ ] Скрытый инструмент (NCSKSPCX2USD) не участвует в анализе.
- [ ] Дизайн неотличим от остального приложения: тёмная тема, карточки, каскад появления, кнопки с откликом.
