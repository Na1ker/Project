# BingX Trade Tracker — трекер и статистика фьючерсных сделок

## Цель (Objective)

На биржах (BingX, Binance) нет нормальной, красивой статистики по собственной торговле. Трейдеру негде удобно посмотреть доходность за период, историю сделки на графике и причины своих входов.

Создаём **личный локальный веб-сервис**, который подключается к BingX по API-ключу, каждую минуту подтягивает данные о фьючерсных сделках, хранит их в собственной базе и показывает красивый дашборд: открытые позиции, статистику за 7/30/90/180 дней, доходность, капитал, а по каждой закрытой сделке — страницу с графиком входа/выхода, PnL, комиссиями и фандингом. К сделкам можно писать заметки о причинах входа.

Пользователь v1 — один человек (владелец), запуск локально на его Mac. Публичность, другие биржи и продвинутая аналитика — следующие версии.

## Требования (Requirements)

### Обязательные (Must-have)

**Подключение и синхронизация**
1. Подключение аккаунта BingX (USDT-M perpetual фьючерсы) по API-ключу и секрету. Ключи вводятся один раз в настройках, хранятся только локально (файл `.env` или локальная база), никогда не попадают в git.
2. Фоновая синхронизация **каждые 60 секунд**: открытые позиции, новые ордера/исполнения, баланс и капитал аккаунта, фандинговые платежи.
3. При первом подключении — загрузка всей доступной истории сделок из API BingX, минимум за 180 дней (под самый длинный период статистики).
4. Повторная синхронизация идемпотентна: уже сохранённые сделки не дублируются (уникальность по ID ордера/исполнения биржи).
5. Из отдельных исполнений (fills) реконструируются **сделки** (round-trip: открытие позиции → полное закрытие), включая случаи усреднения (несколько входов) и частичного закрытия (несколько выходов). Сделка агрегирует: инструмент, направление (long/short), средняя цена входа/выхода, объём, время первого входа и последнего выхода, реализованный PnL, суммарная комиссия, суммарный фандинг за время удержания.

**Хранение**
6. Все данные (сделки, исполнения, снапшоты капитала, фандинг, заметки) хранятся в локальной базе SQLite. База переживает перезапуск приложения; данные не теряются и не требуют повторной загрузки с биржи.
7. Ежедневный (и при каждой синхронизации — внутридневной) снапшот капитала аккаунта для построения кривой капитала.

**Дашборд (главная страница)**
8. Переключатель периода: **7 / 30 / 90 / 180 дней**. Все метрики и графики пересчитываются под выбранный период.
9. Карточки метрик за период: суммарный реализованный PnL (в USDT и % от капитала на начало периода), винрейт, число сделок, профит-фактор, средняя прибыльная/убыточная сделка, суммарные комиссии и фандинг.
10. График кривой капитала за выбранный период.
11. Текущий общий капитал аккаунта (баланс + нереализованный PnL) — виден всегда.
12. Блок открытых позиций: инструмент, направление, объём, цена входа, текущая цена, нереализованный PnL, время в позиции. Обновляется при каждой синхронизации.

**История сделок и страница сделки**
13. Список закрытых сделок с сортировкой по дате закрытия: инструмент, направление, даты входа/выхода, объём, PnL (цвет: прибыль/убыток), наличие заметки. Клик по сделке открывает её страницу.
14. Страница сделки: **свечной график** инструмента за период сделки с запасом до и после (таймфрейм подбирается автоматически по длительности сделки), с маркерами точек входа и выхода (все входы и выходы, если их несколько).
15. Справа от графика — панель фактов сделки в строчном формате: дата/время входа и выхода, объём, средние цены входа/выхода, плечо (если доступно из API), реализованный PnL, суммарная комиссия за сделку, суммарный фандинг за время удержания позиции.
16. Свечи для графика берутся из публичного API BingX и кэшируются в базе, чтобы страница сделки открывалась и без сети.

**Заметки**
17. К любой сделке (открытой или закрытой) можно добавить/отредактировать текстовую заметку «почему зашёл». Заметка сохраняется в базе и отображается на странице сделки и признаком в списке сделок.

**Дизайн**
18. Тёмная тема: почти чёрный фон, карточки со скруглёнными углами, крупная чистая типографика. Акцентная палитра: **тёмно-красный + чёрный** как фирменное сочетание (референс: дашборд в стиле Tomtrik/COT), зелёный — прибыль, красный — убыток. Интерфейс на русском языке.
19. Приложение открывается в браузере локально (например `http://localhost:3000`), одной командой запуска.

### Желательные (Nice-to-have, НЕ входит в v1)

- Монте-Карло симуляция и другие статметоды по накопленным сделкам (тестирование и понимание своей торговли).
- Анализ накопленных причин входа из заметок; наложение истории причин на график/актив как подсказка при торговле.
- Другие биржи (Binance, Bybit и т.д.), мультибиржевой агрегированный портфель.
- Публичный доступ: регистрация, мультипользовательность, хостинг.
- Теги/категории заметок, скриншоты к сделкам.
- Уведомления (Telegram и т.п.).

## Ограничения (Constraints)

- **Биржа:** только BingX, только USDT-M perpetual фьючерсы. Архитектура коннектора — за интерфейсом (interface/abstract class), чтобы другие биржи добавлялись без переписывания ядра.
- **Платформа:** веб-приложение, запуск локально на macOS (Apple Silicon), доступ через браузер.
- **Стек:** Next.js (React, TypeScript) + SQLite (через Prisma или аналог) + фоновый воркер синхронизации в том же процессе/приложении. Графики: TradingView lightweight-charts для свечей, лёгкая chart-библиотека для кривой капитала. (Стек можно заменить при веских причинах, зафиксировав решение в отчёте /build.)
- **API BingX:** соблюдать rate limits (backoff при HTTP 429), подписывать приватные запросы; учитывать ограничение глубины истории у отдельных эндпоинтов — тянуть максимум доступного.
- **Секреты:** API-ключи не коммитятся, не логируются, не отображаются в UI после сохранения (только маска).
- **Время:** все даты в UI — в локальной таймзоне пользователя; в базе — UTC.

## Крайние случаи (Edge cases)

- **Невалидный/просроченный API-ключ** → на дашборде явный баннер «нет соединения с BingX, проверь ключи», приложение не падает, показывает данные из базы.
- **BingX недоступен или rate limit (429)** → синхронизация пропускает цикл с экспоненциальным backoff, в UI виден индикатор «последняя синхронизация N минут назад».
- **Пустая база (первый запуск, ключи ещё не введены)** → экран онбординга с формой ключей, а не пустой дашборд с нулями.
- **Сделок за выбранный период нет** → карточки показывают «нет сделок за период», не NaN/деление на ноль (винрейт, профит-фактор при нуле убыточных сделок — «∞» или прочерк, определить единообразно).
- **Позиция ещё открыта** → в списке сделок не показывается как закрытая; на её странице (из блока открытых позиций) — график с маркером входа, PnL помечен как нереализованный, фандинг/комиссия — накопленные на текущий момент.
- **Усреднения и частичные закрытия** → сделка корректно агрегируется (требование 5); на графике маркеры всех входов/выходов.
- **Сделка старше доступной истории свечей** (или инструмент делистнут) → страница сделки показывает факты без графика с пометкой «график недоступен», не ошибку.
- **Реверс позиции одним ордером** (long → short без паузы) → исполнение делится между закрывающейся и новой сделкой; допустимо упрощение: закрытие старой и открытие новой сделки в один момент времени.
- **Обрыв синхронизации на середине** (выключили ноутбук) → при следующем запуске синхронизация догоняет пропущенное без дублей и дыр.
- **Очень короткая сделка (< 1 мин) или очень длинная (> 30 дней)** → таймфрейм графика подбирается так, чтобы вход и выход были различимы (от 1m до 1d).

## Definition of Done

- [ ] Ввожу API-ключи BingX в настройках → в течение минуты вижу свои реальные открытые позиции и текущий капитал.
- [ ] История моих закрытых сделок загружена минимум за 180 дней и лежит в SQLite; после перезапуска приложения данные на месте без повторной загрузки.
- [ ] Закрываю сделку на BingX → в течение ~1 минуты она появляется в списке закрытых сделок с верным PnL (совпадает с цифрой BingX).
- [ ] Переключаю периоды 7/30/90/180 → метрики (PnL, винрейт, профит-фактор, комиссии, фандинг) и кривая капитала пересчитываются, цифры согласуются с историей сделок.
- [ ] Кликаю на закрытую сделку → открывается страница со свечным графиком, маркерами входа/выхода, и панелью: даты, объём, цены, PnL, комиссия, фандинг.
- [ ] Пишу заметку к сделке → после перезапуска приложения заметка на месте и видна в списке сделок.
- [ ] Выключаю Wi-Fi → приложение показывает данные из базы и индикатор устаревшей синхронизации, ничего не падает.
- [ ] Ввожу заведомо неверный API-ключ → вижу понятный баннер об ошибке, а не крэш или пустой экран.
- [ ] Дизайн соответствует референсу: тёмный фон, карточки, тёмно-красные/чёрные акценты, зелёный/красный PnL; интерфейс на русском.
- [ ] Приложение запускается одной командой и открывается в браузере.
